 - TCPServer 와 TCPClient 가 연속으로 서로 메세지를 주고 받게 처리함
	=> 전송할 값을 키보드로 입력받아 보냄
	=> 클라이언트가 "quit"를 전송하면 서버에서는 해당 클라이언트와의
	   연결만 끊고 종료하지 않음
	=> 연결을 끊을 때 서버쪽에서 "연결이 끊어졌습니다." 라는 메세지를
	   전송하고 끊음.
	=> 클라이언트는 "연결이 끊어졌습니다." 메세지 확인하고 종료함.

[실습문제]
 - GUI 서버/클라이언트 프로그램 작성
	>> 스트림으로부터 읽기용 메소드
		public void receiveMessage()
	>> 스트림 출력용 메소드 따로 작성 
		public void sendMessage()
	>> 프레임 상속 받는 클래스는 실행용 클래스임
		소켓 레퍼런스를 필드로 선언함
		텍스트에리어, 텍스트필드 레퍼런스도 필드로 선언
		main 에서 소켓 객체 생성해서 프레임 생성자로 넘김
	>> 액션이벤트 발생시 입출력용 메소드 실행시키도록 처리함

 - 서버화면
	텍스트에리어 한 개, 아래쪽에 서버종료 버튼 한개
	>> 텍스트에리어에 클라이언트 정보와 전송메세지 출력처리함
	>> 서버종료 버튼 클릭시 서버프로그램 종료함

 - 클라이언트 화면
	텍스트 에리어 한 개 : 전송메세지와 서버가 보낸 메세지 모두 출력
		가운데 배치
	텍스트필드 한개 : 서버로 보낼 메세지 입력용
		아래쪽에 배치
	보내기 버튼 한개 : 클릭시 서버로 메세지 전송됨
			전송될 때 텍스트에리어에도 출력되게 함
		텍스트필드 오른쪽에 배치
	서버와 연결하기 버튼 한개 : 클릭시 서버와 연결됨
		텍스트에리어에 연결됨 메세지 출력되게 함
		위쪽에 배치
	프로그램 끝내기 버튼 한 개 : 클릭시 프로그램 종료함
		서버와 연결하기 버튼 옆에 배치함
